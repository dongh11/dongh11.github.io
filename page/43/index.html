  • 合并数据框用merge，by=‘keyword’ 指定关键词来合并

关于ggplot2的语法：
每个图层包括如下五个组成部分

  • 数据
  • 审美映射（aes）
  • 统计变换（stat）
  • 几何对象（geom） 即所要创建的图表的类型
  • 位置调节：对图层内的元素位置等进行微调，例如设置填充属性，防止图形重合进行的移动，将柱状图堆叠起来等。

layer()函数有很多参数，就都对应于上述的语法要素：
layer(geom, eom_params, stat, stat_params, data, mapping , position)
大多数情况下，为了方便我们可以使用快捷方式，即直接以geom 或stat 开头，因为任何几何对象都有一个默认的统计变换，任何统计变换都有一个默认的几何对象。

p <- ggplot()
p <- p+ geom_point()

geom_point 就说明了geom=point，然后stat 是默认的，位置调节没有，只需要再给定数据和aes就可以完成一个图层的plot了。

统计变换stat对数据进行操作，生成另一个数据集，并用新的数据及来绘图。
比如stat_bin 就会生成包含各区段观测样本数和密度值的数据集，进而再将这个数据集用于绘图。而且用了stat_bin这样的统计变换，就会默认采用直方图(geom_histogram)这种几何对象。

position=‘jitter’ 是对图像进行调节，即若某个位置已经被另一个数据点占据，则所要画的点微微移动到附近的位置，以免重叠。

关于Ruby

Ruby 支持五种类型的变量。
一般小写字母、下划线开头：变量（Variable）。
$开头：全局变量（Global variable）。
@开头：实例变量（Instance variable）。
@@开头：类变量（Class variable）类变量被共享在整个继承链中，且必须初始化后才能在方法定义中使用。
大写字母开头：常数（Constant）。

It is important to prepare your data in such a way that it gives various different machine learning algorithms the best chance on your problem.

Data pre-processing with caret in R

Transforms can be used in two ways

  • standalone: preProcess() function, and applied to a dataset using the predict() function 从训练集中进行transform，应用于其他数据集
  • training: preProcess() 在模型评价时自动应用，preProcess之后pass to the train() function

the preprocessing function will skip over non-numeric data without error

Below is a quick summary of all of the transform methods supported in the method argument of the preProcess() function in caret.

  • “BoxCox“: apply a Box–Cox transform, values must be non-zero and positive.
  • “YeoJohnson“: apply a Yeo-Johnson transform, like a BoxCox, but values can be negative.
  • “expoTrans“: apply a power transform like BoxCox and YeoJohnson.
  • “zv“: remove attributes with a zero variance (all the same value).
  • “nzv“: remove attributes with a near zero variance (close to the same value).
  • “center“: divide values by standard deviation.
  • “scale“: subtract mean from values.
  • “range“: normalize values.
  • “pca“: transform data to the principal components.
  • “ica“: transform data to the independent components.
  • “spatialSign“: project data onto a unit circle.

Larry Wasserman： Rise of the Machines

During my twenty-five year career I have seen Machine Learning evolve from being a collection of rather primitive (yet clever) set of methods to do classifi- cation, to a sophisticated science that is rich in theory and applications.

To avoid becoming irrelevant, we Statisticians need to (i) stay current on research areas in ML and (ii) change our outdated model for disseminating knowledge and (iii) revamp our graduate programs.

Larry mentioned the competence of statisticians and machine learning researchers. The conference culture of ML gives pressure on researchers and pushes this field to move forward faster than the traditional statistics field.

He has worked with association to ML on the following topics: statistical topology, graphical models, semisupervised inference, conformal prediction, and differential privacy.

There are a number of topics that are routinely covered in ML that we rarely teach in Statistics. Examples are: Vapnik-Chervonenkis theory, concentration of measure, random matrices, convex optimization, graphical models, reproducing kernel Hilbert spaces, support vector machines, and sequential game theory. It is time to get rid of antiques like UMVUE, complete statistics and so on and teach modern ideas.
